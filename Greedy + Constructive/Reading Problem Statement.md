# Reading Problem Statement

**[
](https://codeforces.com/blog/entry/62730)**

[How to read problem statements](https://codeforces.com/blog/entry/62730)

* The result of reading the statement is usually pure math model. Imagine you want to tell the problem to someone else. What parts you want to tell?

* Limitations are part of problem statement. Especially small limitations, because for small data you can try all the possibilities.

* Samples & Notes are part of problem statement. After building math model, check that you model working correct on samples.

* Try to find familiar patterns, maybe objects you already know something about

* Try even harder to spot something strange, something you not expecting. That probably will be the cornerstone of the problem.

* Why is this limitation here? How would the problem change if it is not here?
* What is unusual?
If there is some part of the statement you don't like, try to change that to something you like. Start with understanding the object, then simplify it.

* If the model you get is very big, try to split it into some pieces. It would be great if pieces are independent

* Problemsetters do not write random things in statements.
